appointment-encounter-status-conflict

Appointment and encounter status conflict

Pass

Service counts and scheduling quality are separate questions with separate source-of-truth rules.

Naive failure

Filtering completed encounters through appointment status removes a delivered service merely because its linked appointment remains cancelled.

Expected resolution

Count the current completed encounter as service evidence and independently flag the contradictory appointment status for investigation.

duplicate-encounter-versions

Duplicate and corrected encounter versions

Pass

Count the latest version of each source event, then apply its current status.

Naive failure

Counting every row marked completed produces four services. It double-counts a corrected event and preserves an event that was later voided.

Expected resolution

Rank versions deterministically, retain one current row per source event, and count only current completed events. The correct service count is two.

like-for-like-partial-periods

Like-for-like partial-period comparison

Pass

Operational year-over-year comparisons must use equivalent elapsed windows, not a partial current period against a complete historical period.

Naive failure

Including activity after June 15 in either year changes the comparison boundary and makes the apparent trend depend on how much future data happens to be available.

Expected resolution

Represent the as-of boundaries explicitly. Compare April 1 through June 15 in each fiscal year and exclude later June activity from both windows.

unmapped-program-retention

Unmapped programs remain visible

Pass

A failed dimension lookup is a reportable quality state, not permission to discard the source record.

Naive failure

An inner join to the program mapping table silently drops one legitimate completed encounter and reports only half the delivered activity.

Expected resolution

Retain completed encounters through a left join, report mapped and unmapped counts separately, and surface the missing mapping as a quality signal.
